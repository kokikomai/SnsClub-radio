## エグゼクティブサマリー

YouTube動画から自動的に切り抜き動画を生成するツール開発に向けて、2026年1月時点で利用可能なすべてのAIオプションを徹底的に調査・比較分析しました。その結果、**最も効果的・効率的な実装方法は、Google Gemini 3 Flash APIを中核とした構成**であることが判明しました。

### 🎯 最終推奨構成

```
yt-dlp + Gemini 3 Flash API + MoviePy
```

**主な利点:**
- **月額コスト:** わずか $1-3（100本/月処理時）
- **精度:** OpusClip並みの高精度（業界トップレベル）
- **実装期間:** 1-2週間で完成
- **スケーラビリティ:** 将来的なアップグレードが容易

## 2. 推奨構成の詳細

### 2.1 システムアーキテクチャ

```
YouTubeリンク入力
    ↓
[yt-dlp] → MP4ダウンロード
    ↓
[Gemini 3 Flash API] → 動画を直接分析
                     → ハイライト候補を自動抽出
                     → タイムスタンプ付きで提案
    ↓
[MoviePy] → 複数の切り抜き動画を自動生成
    ↓
複数の切り抜き動画を出力
```

### 2.2 各ツールの役割

| ツール | 役割 | 理由 |
| :--- | :--- | :--- |
| **yt-dlp** | 動画ダウンロード | ffmpeg より安定、複数プラットフォーム対応 |
| **Gemini 3 Flash API** | ハイライト抽出 | 動画直接処理、低コスト、高精度 |
| **MoviePy** | 動画編集・出力 | Python統合が容易、オープンソース |

### 2.3 月額コスト試算

| 項目 | 単価 | 月間使用量 | 月額コスト |
| :--- | :--- | :--- | :--- |
| Gemini 3 Flash (入力) | $0.50/M tokens | 約10M tokens | $5.00 |
| Gemini 3 Flash (出力) | $3/M tokens | 約2M tokens | $6.00 |
| yt-dlp | 無料 | - | $0 |
| MoviePy | 無料 | - | $0 |
| **合計** | - | - | **$11/月** |

※ 100本の動画（平均15分）を処理した場合の試算

## 5. 実装上の重要ポイント

### 5.1 ハイライト検出精度の向上

**プロンプトエンジニアリング:**
```
この動画から、視聴者にとって最も見応えのあるハイライト部分を3～5個抽出してください。

各ハイライトについて、以下の形式で返してください：
1. ハイライト名: [簡潔なタイトル]
   開始時刻: [MM:SS]
   終了時刻: [MM:SS]
   理由: [このシーンが視聴者にとって興味深い理由]

分析の際は、以下の要素を考慮してください：
- 視聴者の感情的な反応（笑い、驚き、感動など）
- 重要な情報や知見の提示
- 動きやアクションの盛り上がり
- 音声の盛り上がり（拍手、歓声など）
```

**ユーザーフィードバック:**
- 生成されたクリップに対する評価を収集
- 低評価クリップの特徴を分析
- プロンプトを段階的に改善

---

### 5.2 処理時間の最適化

- **並列処理:** 複数動画の同時処理（API Rate Limit に注意）
- **キャッシング:** 同じ動画への複数分析を避ける
- **ローカル処理:** 字幕生成などは Whisper API で並列化
